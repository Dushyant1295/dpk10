# GitHub Copilot Instruction Document

This document provides guidelines for using GitHub Copilot effectively in this WordPress boilerplate project.

---

## 1. General Guidelines
- Use GitHub Copilot to assist with repetitive coding tasks, boilerplate code, and generating suggestions.
- Always review the code generated by Copilot to ensure it aligns with the project requirements and coding standards.
- Use Copilot to generate:
  - PHP templates for WordPress pages.
  - Twig templates for rendering views.
  - SCSS files for styling.
  - JavaScript files for interactivity.

---

## 2. Creating New Pages
1. **PHP Template**:
   - Use Copilot to create a new PHP file in the `pages` directory.
   - Example:
     ```php
     <?php
     /**
      * Template Name: New Page
      */

     $context = Timber::context();
     $context['post'] = new Timber\Post();

     Timber::render('pages/new.twig', $context);
     ```

2. **Twig Template**:
   - Use Copilot to create a corresponding Twig file in the `templates/pages` directory.
   - Example:
     ```twig
     {% extends "base.twig" %}

     {% block content %}
     <h1>{{ post.title }}</h1>
     <p>{{ post.content }}</p>
     {% endblock %}
     ```

---

## 3. Creating SCSS Files
1. **New SCSS File**:
   - Use Copilot to create a new SCSS file in the `src/css` directory.
   - Example:
     ```scss
     .new-page {
         h1 {
             color: #333;
         }
         p {
             font-size: 16px;
         }
     }
     ```

2. **Import SCSS**:
   - Ensure the new SCSS file is imported in `main.scss`.
   - Example:
     ```scss
     @import 'new-page';
     ```

---

## 4. Adding Images
1. **Image Path**:
   - Place images in the `src/img` directory.
   - Example: `src/img/new-image.jpg`.

2. **Reference in Twig**:
   - Use Copilot to generate the image path in Twig templates.
   - Example:
     ```twig
     <img src="{{ site.theme_uri }}/src/img/new-image.jpg" alt="New Image">
     ```

3. **Reference in SCSS**:
   - Use Copilot to generate the image path in SCSS files.
   - Example:
     ```scss
     .new-page {
         background-image: url('../img/new-image.jpg');
     }
     ```

---

## 5. Creating JavaScript Files
1. **New JS File**:
   - Use Copilot to create a new JavaScript file in the `src/js` directory.
   - Example:
     ```javascript
     document.addEventListener('DOMContentLoaded', () => {
         console.log('New Page JS Loaded');
     });
     ```

2. **Import JS**:
   - Ensure the new JS file is imported in `main.js`.
   - Example:
     ```javascript
     import './new-page';
     ```

3. **Enqueue Script**:
   - Use Copilot to enqueue the script in `functions.php`.
   - Example:
     ```php
     function enqueue_theme_scripts() {
         wp_enqueue_script('theme-main', get_template_directory_uri() . '/dist/js/main.js', [], '1.0', true);
     }
     add_action('wp_enqueue_scripts', 'enqueue_theme_scripts');
     ```

---

## 6. Testing and Deployment
1. **Test Locally**:
   - Use Copilot to generate commands for testing locally.
   - Example:
     ```sh
     npm run dev
     ```

2. **Build for Production**:
   - Use Copilot to generate commands for building production assets.
   - Example:
     ```sh
     npm run build
     ```

3. **Deploy**:
   - Use Copilot to assist with deployment scripts or CI/CD configurations.

---

By following these guidelines, you can effectively use GitHub Copilot to streamline development in this WordPress boilerplate project.
